Глобальные настройки

git config --global user.name "alexsey-sin"					/Alex15071971/
git config --global user.email "your_email@whatever.com"

Клонирование репозитария с удаленного хоста
git clone https://github.com/alexsey-sin/backend_test_homework

Статус каталога
git status

Добавить файлы в репозитарий
git add .		//все или git add --all
git add <file>	//файл

git rm --cached <file>	//вывести вайл из индексирования
В папке можно создать файл .gitignore в котором можно указать файлы не подлежащие индексированию
В .gitignore коментарии с символа решетки #

Сохранение изменений с пометкой
git commit -m "My first commit"

Добавление измененных файлов (обновление) с изменением коментария
при этом новый коммит не создается
git commit --amend -m "Текст вашего комментария"

Отправка локальных изменений на удаленный сервер
git push	//гит помнит откуда мы клонировали репозитарий

Просмотр истории коммитов
git log		//выйти из режима [Q]

git show HEAD 	//# эта команда выведет на экран последний коммит
git show 1234567	// где вместо 1234567 нужно указать первые 7 символов контрольной суммы нужного коммита

Чтобы вернуть папку в состояние нужного коммита
git reset 97a25f7 	//(первые семь символов контрольной суммы нужного коммита)
все сделанные изменения после коммита будут удалены
Можно откатиться назад только для определенного файла
git reset HEAD program.py	//Указатель HEAD означает самый последний коммит
git reset HEAD	//откатили изменения во всех файлах до предыдущего коммита 

Обновить локальный рапозитарий (при этом локальный репо должен быть закоммичен)
git pull

узнать адрес текущего репозитария
git remote -v
origin  https://github.com/alexsey-sin/hw04_tests.git (fetch)
origin  https://github.com/alexsey-sin/hw04_tests.git (push)


Изменить origin адрес репозитория двумя способами:
1) Первый способ задать url репозитория:
использовать команду git remote set-url origin, например:

git remote set-url origin git@github.com:vicman-wi/wi-backend.git

2) Второй способ:
Отредактировать файл .git/config: секция [remote "origin"] параметр - url. 
Эти два способа идентичные. Т.е. вызов команды git remote set-url origin заменит содержимое git конфига.


origin это ваша ветка: ваше собственное репо на GitHub
upstream обычно относится к первоначальному репо, который вы разветвили (fork)

чтобы отслеживать исходный репо, вам нужно добавить еще один пульт с именем upstream
git remote add upstream git://github.com/<aUser>/<aRepo.git>
