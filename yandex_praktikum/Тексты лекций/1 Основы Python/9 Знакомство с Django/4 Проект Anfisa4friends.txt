Проект Anfisa4friends

Дружба — это чудо. Давайте радовать друзей и угощать их мороженым.
Навыки Анфисы помогут реализовать эту идею.

Напишем проект, с помощью которого можно будет угощать друзей мороженым.
Он будет называться «Анфиса для друзей»: Anfisa4friends.

Как это будет работать?

В проекте будет храниться информация о друзьях и о сортах мороженого.

На странице проекта можно будет указать имя друга и мороженое для него.

Анфиса проверит погоду в городе, где живёт этот друг — и выведет на страницу
всю доступную информацию: кому и куда отправлять мороженое, какая погода за
окном у друга и какое мороженое ему будет отправлено.

Придадим Анфисе человеческие черты: научим её занудству. Пусть она дает друзьям
рекомендации — стоит ли в такую погоду есть мороженое.

Самое время вспомнить о гамбургерах: спроектируем, из каких приложений должен
состоять проект «Анфиса для друзей».

    Приложение «Главная страница»: homepage. Отвечает за отображение главной
	страницы.
    
	Приложение «Мороженое»: icecream. Управляет сортами мороженого.
    
	Приложение «Анфиса»: anfisa. Ищет друга в базе, выясняет город, в котором
	тот живёт и запрашивает погоду в этом городе, выбирает рекомендации.

Вся необходимая информация будет храниться в виде списков и словарей. Список
сортов мороженого будет содержать название и описание для каждого сорта,
а в списке друзей для каждого друга будет указано имя и место жительства.

Первое приложение в проекте Anfisa4friends

На старте работы структура проекта выглядит так:

	├── anfisa4friends
	│   ├── settings.py
	│   └── urls.py  
	├── homepage  
	│   ├── views.py
	│   └── urls.py

В директории, названной по имени проекта, anfisa4friends, хранят файлы,
управляющие работой проекта в целом. А на одном с ней уровне лежат папки
приложений, из которых состоит проект.

Сейчас в файловой структуре проекта уже есть директория homepage. Это первое
приложение проекта, оно отвечает за содержимое главной страницы.
