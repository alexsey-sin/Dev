Задачи на Яндекс.Контест

Все практические задания вам предстоит выполнять на платформе
Яндекс.Контест. Время на решение задач не ограничено. Писать код
нужно на языке Python.

Формат входных данных и примеры

Входные данные ваша программа должна считывать из стандартного потока
ввода stdin. В языке Python, например, для этого обычно используется
функция input(). Альтернатива — читать данные из файла input.txt,
доступного в текущей директории.

В блоке «Формат ввода» описаны данные, которые передаются на вход
вашей программе.

В полях «Пример → Ввод» показаны входные данные, аналогичные тем,
что будут переданы в вашу программу при автоматической проверке задания.

см. скрин_2.jpg

Выполнение задания и вывод ответа

Результат работы вашей программы выведите в стандартный поток вывода
stdout. В языке Python, например, для этого обычно используется функция
print(). Альтернатива — вывести результат в файл output.txt в текущей
директории.

В полях «Пример → Вывод» показаны данные, которые ожидаются от вашей
программы при правильной обработке данных, указанных в поле «Пример → Ввод».

см. скрин_3.jpg

Проверяющей системе не важно внутреннее устройство вашей программы:
она смотрит лишь на результаты работы и на потребляемые ресурсы.

Код вашей программы можно отправить на проверку через веб-форму. Другой
вариант — загрузить на платформу файл с вашим кодом.

Перед отправкой убедитесь, что правильно указали язык, на котором вы
написали решение задачи.

см. скрин_4.jpg

У Яндекс.Контеста есть краткая инструкция
(https://yandex.ru/support/contest/examples-stdin-stdout.html)
 по вводу/выводу данных на различных
языках програмирования.
Проверив ваше решение, система сообщит о результате:

    OK — решение прошло все тесты, это успех.
    WA (Wrong Answer) — ваша программа выдала неправильный результат на одном
		из тестов. Вы увидите номер теста и сможете посмотреть его содержимое.
    CE (Complilation Error) — ошибка компиляции. Возможно, вы неправильно
		выбрали язык программирования при отправке решения на проверку.
		Загляните в отчёт и посмотрите, в чём проблема.
    RE (Runtime Error) — ошибка выполнения. Например, это могут быть ошибки
		вида «выход за границы массива» или необработанные исключения.
    TL (Time Limit) — на одном из тестов решение работает слишком долго.
    ML (Memory Limit) — на одном из тестов решение потребляет слишком много памяти.

Подробный список ответов проверяющей системы можно посмотреть здесь.
(https://contest.yandex.ru/errors/)
